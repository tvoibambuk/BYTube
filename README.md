# MyTube — упрощённый мобильный YouTube (Next.js + Supabase)\n\nЭто стартовый MVP мобильной версии "MyTube":\n- Next.js + TypeScript\n- Tailwind CSS\n- Supabase (Auth, Postgres, Storage) — бэкенд без отдельного сервера\n- Функции: регистрация/вход (Supabase), загрузка видео в storage, лента, поиск, страница просмотра с HTML5-плеером\n\nДобавлена страница авторизации: /auth\nПоддерживается:\n- Вход через Google OAuth\n- Вход по e‑mail (magic link)\n\nКак настроить Google OAuth в Supabase (шаги)\n1. Войдите в ваш проект на https://app.supabase.com\n2. Перейдите в Settings → Auth → Providers.\n3. Включите Google provider:\n   - Нажмите "Enable Google".\n   - Supabase покажет redirect URL(ы), обычно вида: `https://your-project-ref.supabase.co/auth/v1/callback`\n4. В консоли Google Cloud Console:\n   - Создайте проект или используйте существующий.\n   - Откройте "APIs & Services" → "Credentials".\n   - Создайте OAuth 2.0 Client ID (Application type: Web application).\n   - В Authorized redirect URIs добавьте тот URL, что дал Supabase (и http://localhost:3000, если нужно для локальной разработки).\n   - Скопируйте Client ID и Client Secret и вставьте в поля Google Provider в Supabase (Settings → Auth → Providers → Google).\n5. В Settings → API скопируйте Project URL и anon key и поместите их в `.env.local`: |  
   ```\n   NEXT_PUBLIC_SUPABASE_URL=https://your-project-ref.supabase.co\n   NEXT_PUBLIC_SUPABASE_ANON_KEY=your-anon-key\n   ```\n6. После настройки вы сможете:\n   - Открыть `http://localhost:3000/auth` и нажать "Войти через Google" — будет выполнен переход на Google и редирект обратно после подтверждения.\n   - Отправлять magic link на email (без пароля) — форма на странице /auth.\n\nПроверка локально\n1. npm install\n2. Создайте `.env.local` со значениями из Supabase (см. выше)\n3. npm run dev\n4. Откройте http://localhost:3000/auth и протестируйте вход\n\nДополнительно\n- Для production-деплоя добавьте в Authorized redirect URIs адрес вашего деплоя (например https://your-site.vercel.app).\n- Если вы хотите хранить пользовательские профили (ник, аватар, канал), добавьте таблицу `profiles` в базе Supabase и связывайте её с user.id.
